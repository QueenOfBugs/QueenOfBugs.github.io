Emacs Lisp 基础

Emacs 是由 C 语言和 Emacs lisp 语言编写的,因此它内置了对 Emacs lisp 语言的支持.也 因为 Emacs lisp 才让 Emacs 具有更多的可能性. 所以学习 Emacs lisp 对理解和掌握 Emacs 有很大帮助. 为了方便,一般我们将 Emacs Lisp 简称为 Elisp.

Xahlee 的Emacs Lisp 基础部分只教学 Elsip 中对于所有 Lisp 语言都通用的部分,比如基 础语法,数学运算,变量,字符串,循环,分支控制(条件语句),编写函数,以及数据结构.

Elisp 关于 Emacs 作为一个编辑器特有的部分,比如移动鼠标,查找文本,切换缓冲区,插入 文本等在 Emacs Lisp 实践 部分.

执行 Elisp

在任意缓冲区中键入 (+ 3 4) ,然后将光标移动到右括号处,按下 M-x eval-last-sexp 或是使用快捷键 CTRL+x CTRL+e, 也可以选中键入的 lisp 代码,然后 M-x eval-region .上面的几种方法都可以执行键入的代码,然后结果会出现在窗口下面的 消息缓冲区(Messages Buffer)

另外,Elisp 也有像 python 一样的交互式解释器, M-x ielm 即可打开交互式解释器.在 交互式解释器中输入上面代码然后回车也可以执行代码.

还有 eval-buffer 可以执行整个当前缓冲区, load-file 可以执行一个文件,它会提示 你输入一个文件名/路径, 还有 eval-defun 可以执行光标所在之处的函数定义(函数定义 的elisp代码必须有良好的索引,不然 emacs 可能会在查找函数时出现问题.因此不建议使 用,同样还有 narrow-to-defun 也会出问题,导致内容丢失,同样不建议使用)

关于 narrow-to-defun 有问题的部分 Xahlee 的文章是 2015年的,Emacs 24.4.1 但是直 到现在我测试时还是有问题.

打印数据

打印数据是我们调试程序过程中必不可少的部分,使用 (message) 函数可以将数据显示在 消息缓冲区中.

(message "Hi Emacs!") 会在消息缓冲区中显示 "Hi Emacs!"

message 也可以打印变量值,如 (message "Her age is %d" 16), 有学过 C 语言的应 该会比较眼熟这种用法,和 printf() 差不多,占位符 %s 表示字符串, %S 表示任意 Lisp 表达式,例如 (message "Her name is %s" "Jessica"), (message "A List : %S" (list 1 2 3 4))
