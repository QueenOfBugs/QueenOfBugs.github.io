Emacs 编码系统

使用不同语言的用户或多或少地创建了许多不同标准的编码系统来表示它们的语言文字。但 Emacs 内部并不使用这些编码系统，它在读取数据时从各种编码系统转变为自己内部的编码 系统，在写入数据时将内部编码系统转变为其他编码系统.

这种编码的转换会在读或写文件时进行.会在给终端发送命令或接收终端输出时进行,也会 在与子进程交换数据时进行.

有了上面的概念,Emacs 的编码系统设置就好理解一点了,不管是我们键盘输入的数据,还是 直接复制粘贴的数据,都是属于其他编码系统,Emacs 在处理这些数据时会将它们转换成其内 部的编码系统(emacs-internal,或者是相等的uft-8-emacs)

如果想要Emacs不进行任何的编码转换就访问某个文件,则可以使用 find-file-literally 命令,使用这个命令打开一个有中文的文件,会发现中文全都无法正常显示,只有英文显示正 常

Emacs 语言环境设置

Emacs 的语言环境是针对使用不同语言,字符集和编码系统的一系列设置,该设置会修改默认 的输入法,字符集和编码系统.想要查看特定的语言环境,可以使用快捷键 C-h L 或者直接使 用命令方式 M-x describe-language-environment. 默认的语言环境是 "English", 因为是 默认支持的英语环境,因此并没有特别的设置,不关心编码问题,不涉及多种环境切换的可以 看看几种 "Chinese-**" 的语言环境设置,

Fontsets

字体集

一个字体通常定义了单个字母表或字母系统的形状.因此,要显示 Emacs 支持的整个脚本范 围需要许多字体的集合. 在 Emacs 里这样的集合叫做 "fontset" (字体集). 一个字体集通 过一个字体的规格列表定义,每个字体规格被分配来处理一套字符代码,并且对于它指定的字 体并未涵盖的字符,可能会退回到另一个字体集.

每个字体集都有一个名字,就像字体一样. 然而,尽管字体都存储在操作系统里,而且可用的 字体名称由操作系统定义,但是字体集是在 Emacs 本身里面定义的. 一旦你定义了一个字体 集,你可以在任何你能使用单个字体的地方,通过指定它的名字在 Emacs 里使用它. 当然了, Emacs 字体集只能使用你的操作系统支持(已安装)的字体.如果在屏幕上的某些字符是以空 的方框或者十六进制代码的形式出现的,这意味着他们使用的字体集没有这些字符的字体.在 这种情况下,或者是字符显示了,但是不是你想要的样子,你可能需要安装额外的字体或者是 修改字体集来使用你的系统里已经安装的特定字体(见下文). 你的操作系统可能有你可以安 装的可选字体;或者你可以安装 GNU Intlfonts 包,该包涵盖了大多数受支持脚本的字 体.1

Emacs 会自动创建三哥字体集: "standard fontsets", "startup fontset" 和 "default fontset". 默认的字体集(default fontset) 最有可能具有各种非 ASCII 字符的字体,并且 是其他两个字体集的默认备用字体集,如果你设置的是默认字体而不是字体集.然而,它不指 定字体系列名,所以如果直接使用它,结果可能会有点随机. 你可以通过使用 -fn 选项启动 Emacs 来指定特定的字符集. 例如: 

emacs -fn fontset-standard

你也可以通过 Font 资源来指定一个字符集.(参见 X Resources 部分).

如果没有指定使用字体集,那么 Emacs 会使用一个 ASCII 字体,并将 fontset-default 作 为字体没有包含的字符的后备选项. 标准的字体集(standard fontset) 仅在显示请求时使 用,尽管它的名字是标准的字体集.

为了显示一个特定字体集的相关信息, 请使用 M-x describe-fontset 命令. 改名令会提示 你输入一个字体集名称,默认值为当前框架使用的字体集名称,然后显示该字体集中字符的所 有子范围以及分配给它们的字体. 要查看 Emacs 在没有特定字体集的会话(这是通常会发生的),请在提示输入时键入 fontset-default <RET> ,或者只是键入 <RET> 来描述当前框架使用的字体集.

一个字体集不必要为每个字符编码指定字体.如果一个字体集没有为一个特定的字符指定字 体,或者如果它指定了一个操作系统中不存在的字体,那么 Emacs 不能正确显示该字符. Emacs 会将该字符显示为一个十六进制编码或空格或空框. (详情查看 glyphless characters.) 或者一个字体集可能会为某些字符范围指定一个字体, 但是你可能不喜欢他们的视觉表现.如果发生这种情况,你可能希望修改修改你的字体集;要 知道如何操作,请查看 修改字体集(Modifying Fontsets).

修改字体集(Modifying Fontsets)

字体集不必总是从头开始创建. 如果只是需要较小的改动,更简单的方法是修改一个现存的 字体集,通常是 fontset-default (默认字体集). 修改 fontset-default 也会影响其它将 其作为后备的字体集,因此这是解决 Emacs 为特定字母系统选择的字体问题的有效方法.

字体集可以通过使用 set-fontset-font 函数修改,指定要修改字体的字符,字符集,字母系 统或者是字符范围,以及要使用的字体的字体规格. 例如: 

;; Prefer a big5 font for han characters.
;; 为 han (汉)字符首选 big5 (繁体)字体
(set-fontset-font "fontset-default"
                  'han (font-spec :registry "big5")
                  nil 'prepend)

;; Use MyPrivateFont for the Unicode private use area.
;; 将 MyPrivateFont 用于 Unicode 专用区域(Unicode 字符范围 e000-f8ff).
(set-fontset-font "fontset-default"  '(#xe000 . #xf8ff)
                  "MyPrivateFont")

;; Use Liberation Mono for latin-3 charset.
;; 为 latin-3 字符集使用 Liberation Mono 字体
(set-fontset-font "fontset-default" 'iso-8859-3
                  "Liberation Mono")

;; Use DejaVu Sans Mono as a fallback in fontset-startup
;; before resorting to fontset-default.
;; 在使用 fontset-default 作为后备之前将 DejaVu Sans Mono 作为 fontset-startup 的后备字体
(set-fontset-font "fontset-startup" nil "DejaVu Sans Mono"
                  nil 'append)

更多关于使用 set-fontset-font 函数的细节请查看 (elisp)Fontsets.

如果你不知道字符的字符码,或者是它属于哪一个字母系统,你可以询问 Emacs. 光标位于字 符处时,键入 C-u C-x = -- (what-cursor-position), 那么此信息以及更多的信息将显示 在 Emacs 弹出的 Help 缓冲区中. 参见 位置信息(Position Info). 例如, 日语字符属于 kana 字母系统,但是日语文本也将其与汉语字符混合了,因此,下面的 代码使用 han 字母系统来设置 Emacs, 让其对日语文本使用 Kochi Gothic 字体: 

(set-fontset-font "fontset-default" 'han "Kochi Gothic")

(为了方便起见, Emacs 中的 han 字母系统设置为支持所有中文,日文和韩文,即 CJK 字符, 而不仅仅是中文字符.)

请参阅变量 script-representative-chars 的文档来查看已知的字母系统.

像上面的字体集设置只影响默认字体不支持的字符,所以如果 Kochi Gothic 字体涵盖了拉 丁字符,它也不会被用来显示拉丁字母系统,因为 Emacs 使用的默认字体通常都涵盖了基本 的拉丁字符.

一些你的操作系统上安装的字体可能已经损怀,或者使用他们的字符产生一些不愉快的结果, 你可能希望指示 Emacs 在搜索显示字符所需要的合适字体时完全忽略这些字体. 你可以通 过将有问题的字体添加到变量 face-ignored-fonts 的值里来实现这一点,该变量是一个列 表. 下面是一个放入到你的 ~/.emacs 文件的示例: 

(add-to-list 'face-ignored-fonts "Some Bad Font")
